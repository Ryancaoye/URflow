# 2026-04-15 Daily Notes

## URflow Bug 修复：曲线守护器（20:20 - 20:35）

### 问题描述
用户在 Animation Window 中通过 URflow 设置好贝塞尔曲线后，一旦在 Dopesheet 或 Curves 模式下拖动关键帧位置，Unity 会内部重建 AnimationCurve，将 `weightedMode` 重置回 `WeightedMode.None`，`inWeight/outWeight` 恢复默认值（1/3），导致自定义曲线形状丢失、回归线性。

### 根因分析
Unity Animation Window 在移动 keyframe 时调用 `AnimationUtility.SetEditorCurve` 重建关键帧数据，不保留 weighted mode 状态。这是 Unity 的行为而非 URflow 的代码缺陷，但需要在 URflow 端做缓解。

### 解决方案：URflowCurveGuard 自动守护器
新增 `[InitializeOnLoad]` 静态守护器类，核心机制：

1. **Apply 后自动注册** — 每次成功应用贝塞尔曲线后，记录 `(clip, binding, 帧索引对)` 和对应的贝塞尔参数 + 当前 tangent/weight 快照
2. **EditorApplication.update 持续监控** — 每 6 帧检测一次（约 10Hz），检查被监控关键帧的 `weightedMode` 和 `weight` 是否被 Unity 重置
3. **检测到重置立即修复** — 用缓存的贝塞尔参数根据帧的**新位置**重新计算 tangent，重新设置 weight 和 weightedMode，自动刷新 Animation Window
4. **支持帧位置变化** — tangent 基于移动后的新 `time/value` 重新计算，曲线形状正确适应新位置

### 性能影响
- 极小。每次检测仅遍历已注册的帧对（通常几对），调用 `AnimationUtility.GetEditorCurve` + 比较 float 值
- `_tracked` 为空时第一行 return，零开销
- 修复操作（`SetEditorCurve` + Undo）只在检测到重置时触发
- 6 帧间隔在编辑器 60fps 下约 10Hz，开销可忽略

### 新增文件
- `URflowCurveGuard.cs` — 曲线守护器，`[InitializeOnLoad]`，通过 `EditorApplication.update` 持续监控

### 修改文件
- `URflowApplyHelper.cs` — Apply 成功后调用 `URflowCurveGuard.Register` 注册监控（在 `SetEditorCurve` 之后重新读取 Unity 存储的实际数据作为快照）

### 关键技术点
- Unity Animation Window 移动 keyframe 会重置 `weightedMode`/`inWeight`/`outWeight` — 这是 Unity 行为非 bug
- `[InitializeOnLoad]` + 静态构造函数注册 `EditorApplication.update` 实现编辑器级后台轮询
- tangent 需要根据新的 time/value 重新计算而非简单恢复旧值，否则曲线形状在帧间距变化时会变形
- `Undo.RecordObject` 确保自动修复操作可撤销
